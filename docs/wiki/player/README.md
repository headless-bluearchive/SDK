# 玩家接口

这里放已经封装成 SDK 方法的游戏内接口。README 只写个大概。

当前文档：

- [账号资源](game-account.md)：货币、资源同步。
- [基础只读接口](game-readonly.md)：学生、主线、日常副本这类只读状态。
- [签到](attendance.md)：普通签到、活动签到。
- [咖啡厅](cafe.md)：咖啡厅状态、摸头、收益领取。
- [任务](mission.md)：任务列表和领奖限制。
- [邮件](mail.md)：邮件检查、列表、领取。
- [课程表](academy.md)：课程表状态和普通课程表执行。
- [MomoTalk](momotalk.md)：未读、对话推进、羁绊剧情。
- [商店](shop.md)：商店列表和 AP 购买。
- [扫荡](sweep.md)：扫荡预设列表。

统一原则也很简单：

- SDK 给稳定的状态和发包方法。
- 状态变更接口会先做基本校验。
- 会消耗资源、会改变账号状态、或者服务器行为还不稳定的地方，会让调用方显式确认。
